Mapbox 样式文件是用来定义地图外观的文件：绘制什么数据、绘制顺序是什么以及绘制时给数据添加什么样式都是样式文件要处理的问题。样式文件是一个带有特定根级和嵌套属性的 JSON 对象。 本样式规范将定义和描述这些属性。

本样式规范的用户对象包括：

  • 想手动编写样式，而非用 Mapbox Studio
    • 的高级设计师和地图绘制人员。
  • 要使用Mapbox GL JS 或 Mapbox Android SDK
    • 相关功能的开发人员
  • 开发生成或处理 Mapbox 样式软件的作者。

使用 Mapbox iOS SDK 或 Mapbox macOS SDK 的开发人员应参考与样式功能相关的 iOS SDK API 平台适用参考文档。

Sources

数据源（Sources）提供显示在地图上的数据。 数据源的类型由 "type" 属性指定， 并且必须是 <%= source_types.join(', ') %>的形式。 添加数据源并不会立即在地图上显示，因为数据源不包括像颜色或宽度这样的细节。不同图层可涉及同一个数据源并为其添加视觉效果。这就让我们可以用不同的方法为同一个数据源添加样式，如在道路图层中区分不同类型的路。

切片数据源（矢量和栅格）必须就 TileJSON specification指定出细节。 有几种方法可以实现：

  • 直接在数据源中使用 TileJSON 属性，如： "tiles", "minzoom", 和 "maxzoom"
    {% highlight json%} "mapbox-streets": { "type": "vector", "tiles": [ "http://a.example.com/tiles/{z}/{x}/{y}.pbf", "http://b.example.com/tiles/{z}/{x}/{y}.pbf" ], "maxzoom": 14 } {% endhighlight %}
  • 给 TileJSON 源添加一个 "url"：
    {% highlight json%} "mapbox-streets": { "type": "vector", "url": "http://api.example.com/tilejson.json" } {% endhighlight %}
  • 给 WMS 服务器添加一个支持 EPSG:3857 (or EPSG:900913) 的 url 作为切片数据的数据源。 服务器 url 应包含一个 "{bbox-epsg-3857}" 替换令牌（replacement token）以提供 bbox 参数。 {% highlight json%} "wms-imagery": { "type": "raster", "tiles": [ 'http://a.example.com/wms?bbox={bbox-epsg-3857}&format=image/png&service=WMS&version=1.1.1&request=GetMap&srs=EPSG:3857&width=256&height=256&layers=example' ], "tileSize": 256 } {% endhighlight %}

{% highlight json%} "mapbox-streets": { "type": "vector", "url": "mapbox://mapbox.mapbox-streets-v6" } {% endhighlight %}

Sprite

样式的 sprite 属性将在渲染 background-pattern, fill-pattern, line-pattern, 和 icon-image 样式属性时为加载的小图提供 URL 模板。 <% if (ref.$root.sprite.example) { %>

{% highlight json %} <%= '"sprite": ' + JSON.stringify(ref.$root.sprite.example, null, 2) %> {% endhighlight %}
<% } %>

有效的 sprite 数据源必须提供两种文件类型：

  • index 文件，即对 sprite 中每一个图像都有描述的 JSON 文件。 该文件的内容必须为一个 JSON 对象且其 key 值构成该文件的标识符，这些标识符将被用作上述样式属性的值且这些值是用来描述图像尺寸 (width 和 height 属性)、像素比 (pixelRatio) 和在 sprite (x 和 y) 中的位置。 例如，包含单个图像的 Sprite 可能具有以下索引文件内容：
    {% highlight json%} { "poi": { "width": 32, "height": 32, "x": 0, "y": 0, "pixelRatio": 1 } } {% endhighlight %}
    然后样式就可以通过 layout 属性 "icon-image": "poi"，或符号化的值 "icon-image": "{icon}" 和带 icon 属性、poi 值的切片要素，创建一个符号图层来参考这个 sprite 图像。
  • 图像文件，即包含 sprite 数据的 PNG 图像。

Mapbox SDKs 将使用样式中 sprite 属性的值生成 URLs 以加载这两个文件。 首先，对两种文件类型它都将在高-DPI 设备的 URL 后添加 @2x。 其次，它将给索引文件添加文件扩展名： .json，为图像文件添加扩展名 .png。 例如，如果你指定了 "sprite": "https://example.com/sprite"，渲染器将加载 https://example.com/sprite.json 和 https://example.com/sprite.png，或 https://example.com/sprite@2x.json 和 https://example.com/sprite@2x.png。

如果你正使用 Mapbox Studio，你将使用 Mapbox 提供的预构建 sprite，或者你可以上传自定义 SVG 图像以构建你自己的 sprite。 无论哪种情形，Mapbox API 将自动构建并提供 sprite。 如果你想手动创建 sprite 并由自己来托管文件，可以用 spritezero-cli，该命令行实用程序会在 SVG 目录建立与 Mapbox GL 兼容的 sprite PNG 和索引文件。

Glyphs

样式的 glyphs 属性为加载 PBF 格式的 signed-distance-field 字形符号（glyph）集提供 URL 模板。 <% if (ref.$root.glyphs.example) { %>

{% highlight json %} <%= '"glyphs": ' + JSON.stringify(ref.$root.glyphs.example, null, 2) %> {% endhighlight %}
<% } %> 该 URL 模板应包含两个令牌（token）：
  • {fontstack} 在请求获取字形时，由某个符号图层 text-font 属性指定的字体堆栈中， 用逗号分隔的字体列表将替换该令牌。
  • {range} 在请求获取字形时，该令牌将被 256 个 Unicode 代码点范围内的点替换。例如，要为 Unicode Basic Latin 和 Basic Latin-1 Supplement blocks加载字形，范围将为 0-255。 实际的加载范围是在运行时根据需要呈现的文本内容而定的。

图层有两个子属性用来定义如何渲染该图层的数据，即 layout 和 paint 属性。

Layout 属性 位于图层的 "layout" 对象中。 在渲染过程之初，这两个属性将被用来定义如何将该图层的数据传递给 GPU。 为了提高效率，图层可以通过 "ref" 与另一图层共享 layout 属性，并在需要的时候尽可能共享。 这样可以减少处理时间并使两个图层可以共享 GPU 内存和其它与图层关联的数据源。

Types

Mapbox 样式包含不同类型的值，最常见的是图层的样式属性值。

Color

颜色可以多种可用的格式写为 JSON 字符串： HTML-style hex values, rgb, rgba, hsl, 和 hsla。 同时允许预定义 HTML 颜色名称，如 yellow 和 blue。

{% highlight json %} { "line-color": "#ff0", "line-color": "#ffff00", "line-color": "rgb(255, 255, 0)", "line-color": "rgba(255, 255, 0, 1)", "line-color": "hsl(100, 50%, 50%)", "line-color": "hsla(100, 50%, 50%, 1)", "line-color": "yellow" } {% endhighlight %}

尤其值得注意的是对 hsl 的支持，它比 rgb() 更易推论。

Enum

字符串值的固定列表之一。 对值使用引号。

{% highlight json %} { "text-transform": "uppercase" } {% endhighlight %}

String

字符串基本上只是文本。 在 Mapbox 样式中，你需要将字符串放在引号里。 字符串可以是任何东西，但需要注意 text-field - 实际上它将参照要素，你需要将要素放在花括号里，如下列示例所示。

{% highlight json %} { "text-field": "{MY_FIELD}" } {% endhighlight %}

Boolean

布尔值（Boolean）表示是或否，因此它接收的值为 true 或 false。

{% highlight json %} { "fill-enabled": true } {% endhighlight %}

Number

为数字值，通常为整数或浮点数（十进制数）。书写时不用引号。

{% highlight json %} { "text-size": 24 } {% endhighlight %}

Array

数组（Arrays）是以特定顺序列出的由一个或多个数字组成的逗号分隔列表。例如，它们在 line dash 数组中使用，其中的数字将指定线的间距、间隔和线。

{% highlight json %} { "line-dasharray": [2, 4] } {% endhighlight %}

Function

Layout 或 paint 属性的任何值都可以指定为 function。 函数（Functions）可以帮助你根据当前的缩放级别和/或要素的属性来改变地图要素的外观。

stops
需要的（除了 identity 函数）数组。
函数根据输入和输出值进行定义。 一组由一个输入值和一个输出值组成的数组称作一个"stop。"
property
也可为 string。
如果指定了，函数将把指定的要素属性作为一个输入值。 查看 Zoom Functions 和 Property Functions 以获取更多信息。
base
也可为 number。 默认是 <%= ref.function.base.default %>。
插值曲线的指数函数的底。它控制函数输出值增加的速率。值越高，输出值就越接近范围的最高值。当值越接近 1，输出值就呈线性上涨。
type
也可为 enum。 One of identity, exponential, interval, categorical.
identity
函数返回它们的输入值作为输出值。
exponential
函数通过在 stop 之间插值生成一个输出值，该值仅刚好小于或大于函数输入值。 域必须是数字。 对于标记为 "exponential" 符号的属性这是默认的。
interval
函数返回 stop 的输出值，该值仅刚好小于或大于函数输入值。 域必须是数字。 对于标记为 "interval" 符号的属性这是默认的。
categorical
函数返回的 stop 输出值等于函数输入值。
default
当值不可用时，该值将作为回退函数的值产生。 该值在以下情况下使用：
  • 在 categorical 函数中，当要素的值和 stop 域中的所有值均不匹配时。
  • 在 property 和 zoom-and-property 函数中，当一个要素不含特定属性值时。
  • 在 identity 函数中，当要素值对样式属性无效时（例如，如果函数被用于 circle-color 属性但要素属性值不是字符串或不是有效的颜色值）。
  • 在 interval 或 exponential property 和 zoom-and-property 函数中，当要素值不是数字时。
如果未提供默认值，样式属性的默认值将被用于这些请求。
colorSpace
也可为 enum。 rgb，lab，hcl之一。
插入颜色值的颜色空间。在如 LAB 和 HCL 的感知颜色空间中插入颜色产生的颜色过渡表会比 RGB 空间中插入的颜色更协调、颜色更易于区分。
rgb
用 RGB 颜色空间插入颜色值
lab
用 LAB 颜色空间插入颜色值。
hcl
用 HCL 颜色空间插入颜色值，分别插入色彩（Hue）、 色度（Chroma）和亮度（Luminance）通道。

Zoom functions 允许地图要素的外观随着地图的缩放级别而改变。 Zoom functions 可以用来创建景深的视觉效果并控制数据的密度。 每一个 stop 都是有两个元素的数组： 第一个是缩放级别，第二个是一个函数输出值。

{% highlight js %} { "circle-radius": { "stops": [ // zoom is 5 -> circle radius will be 1px [5, 1], // zoom is 10 -> circle radius will be 2px [10, 2] ] } } {% endhighlight %}

color，number 和 array 属性的渲染值都被插入在了不同的 stop 之间。 Enum，boolean 和 string 属性值无法插入，因此它们的渲染值只有在指定的 stop 中被改变。

Zoom functions 在渲染 layout 和 paint 属性时有重要的区别。 任何时候缩放级别发生改变, 即使是分数大小的变化，Paint 属性都会持续重新求值。 Paint 属性的渲染值将发生变化，例如，当地图在缩放级别 4.1 和 4.6 之间进行变化时。 而另一方面 Layout 属性仅在每一个整数缩放级别时求值。 接着上一个例子，layout 属性的渲染将 不会 在缩放级别位于 4.1 和 4.6 之间时改变，无论指定的是哪些 stop； 但缩放级别为 5时，该函数将重新求值，且属性的渲染值将会改变。（你可以在 layout 属性缩放函数中加入分数缩放级别，这会影响生成的值；但是渲染仍然只会在整数缩放级别时发生改变。）

Property functions 允许地图元素的外观随着其属性而改变。 Property functions 可以被用来在同一图层从视觉上区分要素类型或创建可视化。 每一个 stop 都是由两个元素组成的数组，一个属性输入值和一个函数输出值。 注意此时在所有属性和平台上都不能获取对属性函数的支持。

{% highlight js %} { "circle-color": { "property": "temperature", "stops": [ // "temperature" is 0 -> circle color will be blue [0, 'blue'], // "temperature" is 100 -> circle color will be red [100, 'red'] ] } } {% endhighlight %}

Zoom-and-property functions 允许地图要素的外观跟随其属性 和 缩放级别而改变。 每一个 stop 都是由两个元素组成的数组，带一个属性输入值和一个缩放值的对象，和一个函数输出值。 注意对属性函数的支持尚未完成。

{% highlight js %} { "circle-radius": { "property": "rating", "stops": [ // zoom is 0 and "rating" is 0 -> circle radius will be 0px [{zoom: 0, value: 0}, 0], // zoom is 0 and "rating" is 5 -> circle radius will be 5px [{zoom: 0, value: 5}, 5], // zoom is 20 and "rating" is 0 -> circle radius will be 0px [{zoom: 20, value: 0}, 0], // zoom is 20 and "rating" is 5 -> circle radius will be 20px [{zoom: 20, value: 5}, 20] ] } } {% endhighlight %}

Filter

筛选器（filter）从图层选择特点的要素。 筛选器是以下形式之一：

Existential Filters

["has", key] feature[key] 存在
["!has", key] feature[key] 不存在

Comparison Filters

["==", key, value] 相等： feature[key] = value
["!=", key, value] 不相等： feature[key] ≠ value
[">", key, value] 大于： feature[key] > value
[">=", key, value] 大于或等于： feature[key] ≥ value
["<", key, value] 小于： feature[key] < value
["<=", key, value] 小于等于： feature[key] ≤ value

Set Membership Filters

["in", key, v0, ..., vn] set inclusion: feature[key] ∈ {v0, ..., vn}
["!in", key, v0, ..., vn] set exclusion: feature[key] ∉ {v0, ..., vn}

Combining Filters

["all", f0, ..., fn] logical AND: f0 ∧ ... ∧ fn
["any", f0, ..., fn] logical OR: f0 ∨ ... ∨ fn
["none", f0, ..., fn] logical NOR: ¬f0 ∧ ... ∧ ¬fn

key 必须是识别要素属性的字符串，或是以下特殊 key 之一：

  • "$type"： 要素类型。 该 key 可以和 "==", "!="、 "in" 和 "!in" 运算符。 可能的值为 "Point"、 "LineString"、 和 "Polygon"。
  • "$id"： 要素的识别标记（identifier）。 该值可以和 "=="、 "!="、 "has"、 "!has"、 "in"、 和 "!in" 运算符使用。

value（和 v0, ..., vn 集合算符） 必须是 string、 number 或 boolean 之一用以比较属性值。

Set membership filters 是测试某个字段是否与多个值中的任何一个匹配的紧凑且有效的方法。

Comparasion 和 set membership filters 执行强类型（strictly-typed）的对比； 例如，所有以下值均为 false： 0 < "1", 2 == "2", "true" in [true, false].

"all"、"any" 和 "none" 过滤器运算符用来创建 compound 过滤器。 f0, ..., fn 必须是过滤器表达式。

{% highlight json%} ["==", "$type", "LineString"] {% endhighlight %}

该过滤器要求每个要素的 class 属性等于 "street_major"、 "street_minor"、 或 "street_limited"。

{% highlight json%} ["in", "class", "street_major", "street_minor", "street_limited"] {% endhighlight %}

组合过滤器（combining filter）"all" 使用跟随它的其它三个过滤器并要求它们对某个将被包括的要素全部为 true： 此要素必须有一个 class 等于 "street_limited"，其 admin_level 必须大于等于 3，且其类型不能为多边形（Polygon）。 你可以把组合过滤器改为 "any" 以允许要素匹配即将包含的任意标准 —— 要素为多边形（Polygons），但是有不同的 class 值等等。

{% highlight json%} [ "all", ["==", "class", "street_limited"], [">=", "admin_level", 3], ["!in", "$type", "Polygon"] ] {% endhighlight %}
